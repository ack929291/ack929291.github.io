> “介绍一下你自己”
> “你好！我是Qwen，是由阿里云开发的超大规模语言模型。”

很多人听说过神经网络、权重、预测token，或者听说过Transformer、Attention，但是无法和物理上LLM如何真正计算和运行联系起来。本系列文章将以 vLLM 和 Qwen2.5-3B 为例，探索运行过程中的环节、当时的设计思路和背后的原理。

# 下载的模型中有什么
```
blobs  refs  snapshots  trees
```
从 Huggingface 下载 Qwen2.5-3B 后，模型目录下有4个目录。`blobs`是真实文件，用哈希命名，内容寻址，用于去重。`refs`是版本引用，记录 main 分支指向哪个快照。`snapshots`是快照目录，也就是Huggingface上看到的文件。`trees`是Xet 存储的目录树结构。
`blobs`是全量文件，`refs`表明现在引用的版本，`snapshots`是当前版本的文件，是`blobs`对应文件的链接。

#### 看起来这里做了版本控制，和git类似，那为什么不直接用git呢？
Huggingface 早期也是用git的，后来换成现在这种形式了。
- Git存的是每个文件每次变更的完整文件，对几KB的代码不是问题，但对动辄几GB的模型文件`model.safetensors`，改动几次后存储会膨胀至几百GB。
- Git 是按文件名和版本存的，Qwen2.5-3B 和 Qwen2.5-7B 共享的同一个 tokenizer.json，在Git里要存两份，而在Huggingface上按 SHA256 内容哈希去重，跨模型跨仓库全局去重。
- Git clone 会把所有历史版本都拉下来，如果模型迭代十几个版本，clone 一次起码要几百GB。Huggingface 可以只下需要的版本（默认只下 main），历史版本按需获取。


```
config.json
generation_config.json
merges.txt
model-00001-of-00002.safetensors
model-00002-of-00002.safetensors
model.safetensors.index.json
tokenizer.json
tokenizer_config.json
vocab.json          
```
这是`snapshots`目录里的文件，也就是模型本身。
## 模型权重
```
model-00001-of-00002.safetensors    ← 权重分片 1
model-00002-of-00002.safetensors    ← 权重分片 2
model.safetensors.index.json        ← 分片索引（告诉程序每个参数在哪个分片里）
```
受文件系统限制，单个文件过大的话不便于传输和加载，此处把3B模型约6GB的权重拆成了两个文件。

## 分词器（Tokenizer）
```
tokenizer.json           ← 分词器的完整定义（首选，fast tokenizer）
tokenizer_config.json    ← 分词器的配置（特殊 token、模板等）
vocab.json               ← 词表（token → ID 的映射）
merges.txt               ← BPE 合并规则
```

## 模型架构配置
```
config.json
```
定义模型"长什么样"：多少层、隐藏维度多大、几个注意力头等。vLLM/transformers 靠它来构建模型结构、分配显存。
里面的内容类似于：
```
{
  "architectures": ["Qwen2ForCausalLM"],
  "hidden_size": 2048,              // 隐藏层维度
  "num_hidden_layers": 36,          // Transformer 层数
  "num_attention_heads": 16,        // 注意力头数
  "num_key_value_heads": 2,         // GQA 的 KV 头数（比 Q 头少 → 节省 KV cache）
  "intermediate_size": 11008,       // FFN 维度
  "num_hidden_layers": 36,
  "vocab_size": 152064,             // 词表大小
  "max_position_embeddings": 32768, // 最大上下文长度
  "torch_dtype": "bfloat16",        // 训练时的精度
  ...
}
```

## 生成配置
```
generation_config.json
```
定义推理生成时的默认参数，比 config 多一层"生成行为"的配置：
```
{
  "do_sample": true,
  "eos_token_id": 151645,        // 结束符 ID（对应 <|im_end|>）
  "max_new_tokens": ...,          // 每次最多生成多少 token
  "temperature": 0.7,
  "top_p": 0.8,
  "top_k": 20,
  "transformers_version": "4.x"
}
```
调用 API 时如果不传这些参数，就用这里的默认值。
